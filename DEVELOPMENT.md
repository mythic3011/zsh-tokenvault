# Development Guide

This guide helps contributors understand the TokenVault codebase and how to extend it.

## Project Structure

```
tokenvault/
├── tokenvault.plugin.zsh      # Plugin loader and entrypoint
├── lib/                        # Shared modules
│   ├── config.zsh             # Configuration, paths, colors, env vars
│   ├── core.zsh               # Core utilities, crypto, file I/O
│   ├── auth.zsh               # Authentication (unlock/lock/unsafe)
│   ├── ui.zsh                 # User interaction (prompts, menus)
│   ├── models.zsh             # Model fetching from APIs
│   ├── prompt.zsh             # Prompt rendering and worker process
│   ├── i18n.zsh               # Internationalization
│   └── completion.zsh         # Tab completion
├── commands/                   # Command implementations
│   ├── profile-commands.zsh   # tv-add, tv-remove, tv-list, tv-help
│   ├── model-commands.zsh     # tv-model-set, tv-model-list, tv-codex-sync
│   └── runtime-commands.zsh   # tv-run, tv-dash
├── build.sh                    # Build and test orchestration
├── test.sh                     # Functional test suite
├── Dockerfile.test             # Docker test environment
├── README.md                   # User documentation
├── INSTALL.md                  # Installation guide
├── CHANGELOG.md                # Version history
└── DEVELOPMENT.md              # This file
```

## Architecture

### Module Loading

The plugin loader (`tokenvault.plugin.zsh`) sources modules in this order:

1. **config.zsh** - Sets up paths, colors, and environment variables
2. **i18n.zsh** - Loads translation strings
3. **core.zsh** - Loads core utilities and crypto functions
4. **models.zsh** - Loads model fetching functions
5. **ui.zsh** - Loads user interaction helpers
6. **prompt.zsh** - Loads prompt rendering and worker
7. **auth.zsh** - Loads authentication functions
8. **completion.zsh** - Loads tab completion
9. **commands/*.zsh** - Loads all command implementations

Each module has:
- Guard against double sourcing: `[[ -n "${TV_*_LOADED:-}" ]] && return 0`
- Lifecycle hooks: `tv_*_open()` and `tv_*_close()` (called by loader)

### State Management

**Encrypted State:**
- `$TV_VAULT` - Encrypted API keys (AES-256-CBC + PBKDF2)
- Master key: `$_TV_MASTER_KEY` (in memory only, unless unsafe mode)

**Unencrypted State:**
- `$TV_PROFILES` - Profile metadata (JSON)
- `$TV_MODELS` - Model defaults per provider (JSON)
- `$TV_USAGE_LOG` - Usage log (JSONL)

**Cache:**
- `$TV_PROMPT_CACHE` - Rendered prompt output
- `$TV_WORKER_LOCK` - Worker process lock (directory)

### Encryption

```zsh
# Encrypt
_tv_crypto enc "json_string"  # Writes to $TV_VAULT

# Decrypt
_tv_crypto dec                # Reads from $TV_VAULT, outputs JSON
```

Uses:
- Algorithm: AES-256-CBC
- Key derivation: PBKDF2
- Salt: Generated by openssl
- Password: `$_TV_MASTER_KEY`

### Async Operations

The `_tv_spawn_worker` function runs in background:

```zsh
_tv_spawn_worker() {
    mkdir "$TV_WORKER_LOCK" 2>/dev/null || return 0  # Acquire lock
    (
        trap 'rm -rf "$TV_WORKER_LOCK"' EXIT          # Release lock
        # ... do work ...
        _tv_write_json "$TV_PROFILES" "$updated"      # Write state
    ) &!                                               # Background job
}
```

Worker tasks:
- Check quota via API
- Update profile status
- Render prompt cache

## Adding a New Command

### 1. Create Command File

Create `commands/mycommand-commands.zsh`:

```zsh
# Guard against double sourcing
[[ -n "${TV_MYCOMMAND_LOADED:-}" ]] && return 0
typeset -g TV_MYCOMMAND_LOADED=1

tv-mycommand() {
    # Require vault to be unlocked
    [[ -z "$_TV_MASTER_KEY" ]] && {
        _tv_print "  ${_TV_RED}✗ Run tv-unlock first${_TV_RST}"
        return 1
    }

    # Your command logic here
    _tv_banner "My Command"
    _tv_print "  ${_TV_GRN}✓ Done${_TV_RST}"
}
```

### 2. Add to Plugin Loader

The loader automatically sources all `commands/*.zsh` files:

```zsh
for tv_command_file in "$TV_PLUGIN_COMMANDS_DIR"/*.zsh(N); do
    source "$tv_command_file"
done
```

### 3. Add Tab Completion

Update `lib/completion.zsh`:

```zsh
_tokenvault() {
    local cmd="$1"
    case "$cmd" in
        tv-mycommand)
            _arguments \
                '-Flag1[Description]:(option1 option2)' \
                '-Flag2[Description]:file:_files'
            ;;
    esac
}

# Register completion
compdef _tokenvault tv-mycommand
```

### 4. Add Help Text

Update `tv-help` in `commands/profile-commands.zsh`:

```zsh
_tv_print "  ${_TV_CYA}tv-mycommand${_TV_RST}                    My command description"
```

### 5. Add Tests

Update `test.sh`:

```zsh
# Test your command
tv-mycommand -Flag1 value >/dev/null
[[ $? -eq 0 ]] || exit 1
```

## Adding a New Module

### 1. Create Module File

Create `lib/mymodule.zsh`:

```zsh
# Guard against double sourcing
[[ -n "${TV_MYMODULE_LOADED:-}" ]] && return 0
typeset -g TV_MYMODULE_LOADED=1

# Your functions here
_tv_my_function() {
    # Implementation
}

# Lifecycle hooks
tv_mymodule_open() {
    [[ -n "${TV_MYMODULE_OPENED:-}" ]] && return 0
    # Initialization code
    TV_MYMODULE_OPENED=1
}

tv_mymodule_close() {
    TV_MYMODULE_OPENED=""
}
```

### 2. Add to Plugin Loader

Update `tokenvault.plugin.zsh`:

```zsh
[[ -f "$TV_PLUGIN_LIB_DIR/mymodule.zsh" ]] && source "$TV_PLUGIN_LIB_DIR/mymodule.zsh"

# ... later ...

if typeset -f tv_mymodule_open &>/dev/null; then
    tv_mymodule_open
fi
```

### 3. Add Tests

Update `build.sh`:

```zsh
typeset -f tv_mymodule_open >/dev/null
```

## Common Patterns

### User Input

```zsh
# Simple prompt
_tv_ask varname "Prompt text" "default_value"

# Secret input (password)
_tv_ask varname "Password" "" "1"

# Menu selection
_tv_menu varname "Choose one" 1 \
    "option1" "description" \
    "option2" "description"
```

### JSON Operations

```zsh
# Read JSON
local data=$(cat "$TV_PROFILES")

# Query JSON
local value=$(echo "$data" | jq -r '.key.subkey // "default"')

# Update JSON
local updated=$(echo "$data" | jq --arg key "value" '.mykey = $key')

# Write JSON
_tv_write_json "$TV_PROFILES" "$updated"
```

### Error Handling

```zsh
# Validate input
_tv_validate_id "$profile_id" || return 1

# Check file exists
[[ -f "$TV_VAULT" ]] || { _tv_print "  ${_TV_RED}✗ Vault not found${_TV_RST}"; return 1; }

# Check command exists
command -v jq >/dev/null 2>&1 || { echo "jq not found" >&2; exit 1; }
```

### Temporary Files

```zsh
# Create temp file with secure permissions
local tmp
tmp=$(_tv_mktemp "$TV_DIR/.json_tmp.XXXXXX") || return 1
chmod 600 "$tmp"

# Use temp file
echo "$content" > "$tmp"

# Atomic move
mv -f "$tmp" "$TV_PROFILES" || { rm -f "$tmp"; return 1; }
```

### Async Operations

```zsh
# Spawn background worker
_tv_spawn_worker

# Wait for worker to complete
wait_for_worker() {
    local i
    for i in {1..50}; do
        [[ ! -e "$TV_WORKER_LOCK" ]] && return 0
        sleep 0.1
    done
    return 1
}
```

## Code Style

### Naming Conventions

- **Public functions**: `tv-command` (commands), `tv_module_open` (lifecycle)
- **Private functions**: `_tv_helper` (internal utilities)
- **Variables**: `TV_CONSTANT` (config), `_TV_PRIVATE` (internal state)
- **Local variables**: `local_var` (lowercase with underscores)

### Zsh Features

- Use `[[ ]]` for conditionals (not `[ ]`)
- Use `(( ))` for arithmetic
- Use `${var:offset:length}` for substring (1-indexed)
- Use `${var//pattern/replacement}` for substitution
- Use `${(P)var}` for indirect variable access
- Use `${var:-default}` for default values

### Error Handling

```zsh
# Always check return codes
command || { _tv_print "error"; return 1; }

# Use set -e in subshells
(
    set -euo pipefail
    # commands here
)
```

### Comments

```zsh
# Use comments for non-obvious logic
# Explain the "why", not the "what"

# Good
# Acquire lock to prevent concurrent modifications
mkdir "$TV_WORKER_LOCK" 2>/dev/null || return 0

# Bad
# Make directory
mkdir "$TV_WORKER_LOCK" 2>/dev/null || return 0
```

## Performance Considerations

### Avoid Subshells

```zsh
# ✗ Slow - creates subshell
data=$(cat "$TV_PROFILES" | jq '.key')

# ✓ Fast - no subshell
data=$(jq '.key' "$TV_PROFILES")
```

### Minimize External Calls

```zsh
# ✗ Slow - multiple jq calls
key1=$(echo "$data" | jq '.key1')
key2=$(echo "$data" | jq '.key2')

# ✓ Fast - single jq call
keys=$(echo "$data" | jq -r '.key1, .key2')
```

### Cache Results

```zsh
# ✗ Slow - recalculates each time
for profile in $(jq -r 'keys[]' "$TV_PROFILES"); do
    # ...
done

# ✓ Fast - cache in variable
local profiles=$(jq -r 'keys[]' "$TV_PROFILES")
for profile in $profiles; do
    # ...
done
```

## Security Considerations

### Input Validation

```zsh
# Always validate user input
_tv_validate_id "$user_input" || return 1

# Sanitize strings
local safe_input="${user_input//[^a-zA-Z0-9_-]/}"
```

### Avoid eval

```zsh
# ✗ Dangerous - eval injection
eval "var=$user_input"

# ✓ Safe - indirect assignment
printf -v var '%s' "$user_input"
```

### File Permissions

```zsh
# Always set restrictive permissions
chmod 600 "$TV_VAULT"
chmod 700 "$TV_DIR"
```

### Sensitive Data

```zsh
# Never log passwords or keys
# ✗ Bad
_tv_print "Key: $api_key"

# ✓ Good
_tv_print "Key: $(_tv_short_key "$api_key")"
```
